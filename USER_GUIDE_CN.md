# Benchmark by CrossDriver — 用户说明书（中文）

这是一份面向使用者的说明书，重点说明“每一项测试是什么、有什么意义、该怎么看”。

## 一、快速开始

1. 打开应用。
2. 在 **Disk** 选择目标磁盘。
3. 选择场景并点击 **Run All**。
4. 等待状态显示 **Completed**。

默认场景为 **Video Creator**。

## 二、场景说明

### Code Editor
- 数据集规模：100,000 文件
- 适合评估：代码工程、小文件密集与目录/元数据操作

### File Editor
- 数据集规模：10,000 文件
- 适合评估：文档编辑、常规文件读写与更新

### Video Creator
- 数据集规模：10,000 文件
- 适合评估：媒体/内容创作下的吞吐与随机 I/O 表现

## 三、每项测试的说明与意义

### 1) Small-file Create Speed（files/s）
- **是什么**：单位时间内可创建多少小文件。
- **意义**：反映代码拉取、依赖安装、缓存生成、解压等“小文件爆发”场景效率。
- **怎么看**：数值越高越好。

### 2) Folder Scan Throughput（files/s）
- **是什么**：对主数据集进行递归遍历时的扫描吞吐。
- **意义**：影响目录浏览、搜索索引、文件树刷新等体验。
- **怎么看**：数值越高越好。

### 3) Metadata Create（ops/s）
- **是什么**：元数据密集操作（如 create/stat/delete）吞吐。
- **意义**：体现文件系统在“目录项与属性操作”上的处理能力。
- **怎么看**：数值越高越好。

### 4) File Edit（ops/s）
- **是什么**：对已有文件进行随机原位编辑的吞吐。
- **意义**：对应频繁小改动、增量保存、补丁式更新等场景。
- **怎么看**：数值越高越好。

### 5) Random Option（Write / Write+fsync）（IOPS）
- **是什么**：随机写入在“普通写”与“写后强制落盘（fsync）”两种模式下的 IOPS。
- **意义**：体现性能与数据落盘安全语义之间的成本差异。
- **怎么看**：两项都越高越好；`Write+fsync`更接近保守耐久写入场景。

### 6) IOPS Stability Curve（1分钟）
- **是什么**：持续负载下 mixed/read/write IOPS 的时间序列曲线。
- **意义**：观察是否稳定、是否随时间衰减（热衰减/缓存影响/竞争）。
- **怎么看**：曲线越平稳、波动越小通常越好。

## 四、综合分（Composite Scores）怎么理解

### Interactive Score
- **含义**：偏“交互响应”的综合能力。
- **来源**：create + folder scan + metadata + file edit 加权融合。
- **适用**：代码/文档/日常文件操作体验评估。

### Media Score
- **含义**：偏“吞吐与持续负载”的综合能力。
- **来源**：顺序吞吐 + 随机写行为加权融合。
- **适用**：媒体素材处理、内容生产、吞吐敏感流水线评估。

## 五、对比结果怎么解读

- 对比页面中的趋势以 `x` 倍数显示（如 `1.25x`）。
- 建议只比较“同一场景 + 相近系统负载”的两次结果。
- 分数是相对指标，主要看 A/B 趋势，不建议当作跨设备绝对排名。

## 六、导出与分享

- **Save Snapshot**：保存主界面当前窗口截图。
- **Export Comparison Image**：导出完整对比图，适合报告/官网/社媒。

## 七、结果更稳定的实践建议

- 测试前关闭重负载后台任务。
- 比较时保持同一磁盘、同一场景。
- 建议连续跑 2–3 次，参考中位趋势而非单次峰值。
